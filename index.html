Instructions:

  • This is a simulation of Pacman moving on in a grid, of dimensions 5 units x 5 units.
  • Pacman is free to roam around the surface of the grid, but must be prevented from moving off the grid. Any movement that would result in Pacman moving off the grid must be prevented, however further valid movement commands must still be allowed.
  • User must enter their commands on the provided text area.
  • Acceptable commands for this application are:
    • PLACE X,Y,F
    • MOVE
    • LEFT
    • RIGHT
    • REPORT
  • The commands are not case-sensitive, but each command must be followed by a new line. Empty lines will be ignored by the application. For example:

    This set of commands is acceptable:
    • place 1,2,NORTH

      MovE
      RIGHT

      place 2,7,EAST
      REPORT

      place 3,2,WEST


      MoVe
      REPORT

    This set of commands is not acceptable:
    • PLACE 1,2,EAST MOVE LEFT REPORT

  • There should be only one command per line and it is not repeated. For example: This set of commands is acceptable:
    • PLACE 1,1,EAST

      MOVE
      LEFT

      REPORT

      MOVE
      REPORT

    This set of commands is not acceptable:
    • PLACE 1,1,EAST

      MOVE MOVE MOVE
      LEFT

      REPORT REPORT

      RIGHT
      REPORT

  • PLACE will put the Pacman on the grid in position X,Y and facing NORTH,SOUTH, EAST or WEST. Any other direction will be ignored by the application. Player can choose any combination of X and Y from (0,0) to (4,4). Any invalid PLACE command will be displayed as a warning by the application, but it will continue to readjusted the sequence.
  • The origin (0,0) can be considered the SOUTH WEST most corner.
  • PLACE command can be called more than once. Pacman's position will be readjusted based on the new PLACE command. For example, if Pacman's position is (2,4) facing NORTH and PLACE 1,1,WEST is called, Pacman's position will be (1,1) facing WEST. All commands after that will be run based on Pacman's (1,1) facing WEST, not the older position.
  • The first valid command to Pacman is a PLACE command. After that, any sequence of commands may be issued, in any order, including another PLACE command. The application should ignore all commands in the sequence until a valid PLACE command has been executed.
  • MOVE will move Pacman one unit forward in the direction it is currently facing. For example, if Pacman's current position is (1,2) facing EAST, MOVE command will move Pacman to (2,2) facing EAST.
  • LEFT and RIGHT will rotate Pacman 90 degrees in the specified direction without changing the position of Pacman. For example, if Pacman is facing NORTH, LEFT command will change Pacman's facing direction to WEST. If Pacman is facing EAST, RIGHT command will change Pacman's facing direction to SOUTH.
  • REPORT will display the current position and facing direction of Pacman. REPORT can be called more than once.
  • Pacman that is not on the grid can choose the ignore the MOVE, LEFT, RIGHT and REPORT commands.
  • Pacman must not move off the grid during movement. This also includes the initial placement of Pacman.
  • Any move that would cause Pacman to move off the grid will be ignored by the application.
